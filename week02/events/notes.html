    1. Local Storage:

      • Check out this site.
      • Local storage: data that persists even if you refresh the page or close the browser.
        Session storage: data that will get cleared when the browser is closed.
      • Storage capcities:
        Local Storage = 5MB
        Cookie storage = 4KB
      • Using local storage you can:
        Set key:value pairs.
        Only store strings.
        Arrays and objects can be stored but you must first convert them to string using JSON.stringify() and JSON.parse().
      • Local storage methods
        setItem(key,value) - add key & value to local storage.
        getItem(keyName) - retrieve a value by the key.
        removeItem(keyName) - remove a value by the key.
        clear() - clear all storage.
        Examples:
        localStorage.setItem('name','Pedro')
        sessionStorage.setItem('name','Side Thai')
        localStorage.getItem('name')
      • To see your local storage, open developer console and select Storage tab.

    2. Programming Basics

      • Template Literals:
         By using backticks, one can manipulate strings as normally but with the added benefit of string interpolation. For this reason, it is not uncommon to see backticks used to create all strings in ES6 code.
        Ex:`My name is ${myName}`;
      • Primitive Datatypes:
        1. String
        2. Symbol
        3. Number
        4. Boolean
        5. Undefined
        6. Null
        Any value that isn't one of these primitive datatypes is an object(functions, arrays, object literals).
      • Only 9 values are always false and these are known as FALSY values:
        1. " " - double quoted empty string literal
        2. ' ' - single quoted empty string literal
        3. ` ` - empty template literal
        4. 0 - zero
        5. -0 - negative zero. Considered different than 0 to JavaScript
        6. NaN - Not a number
        7. false
        8. NULL
        9. Undefined
