    1. TRANSFORMS AND TRANSITIONS

      • TRANSLATE:
        Transforms don't work on inline elements, such as <span>, do you'll need to change the display property to display: inline-block;. Example:

        .ad-ad2 h1 span { font-size: 30px; color: #999999; display: inline-block;}
.ad-ad2 h1:hover span {
  color: #484848;
  transform: translateX(40px);
}
        translate() can be written for both X and Y coordinates, such as transform: translate(45px, -45px) or the coordinates can be adjusted individually with translateX() and translateY(), like so transform: translateX(45px) or transform: translateY(-45px).

      • SCALING:
        The scale(x,y) function scales an element by the defined factors horizontally then vertically. If only one value is provided, it will be used for both the x and y values, growing or shrinking your element or pseudo-element while maintaining the original aspect ratio. For example, scale(1) would leave the element the same size, scale(2) would double its proportions, scale(0.5) would halve them, and so on. You can adjust each coordinate individually with scaleX() and scaleY(), like so transform: scaleX(1.5) or transform: scaleY(2).

        When adding multiple transforms to an element, you shouldn't declare multiple transforms. Simply add the functions to one transform line, like so:

        .ad-ad2 h1:hover span {
color: #484848;
transform: translateX(40px) scale(1.5);
}

      • ROTATION:
        The rotate() function rotates an element around the point of origin by a specified angle value. As with scale, by default the point of origin is the element’s center. Generally, angles are declared in degrees, with positive degrees moving clockwise and negative moving counterclockwise.

        transforms are applied in the order provided.

        .ad-ad2 h1:hover span {
color: #484848;
transform: rotate(10deg) translateX(40px) scale(1.5);
}
        In this case, the span will be rotated 10 degrees, and then moved 40px along the rotated x axis.

      • SKEW:
        The skew(x,y) function specifies a skew along the x and y axes. If the second parameter is omitted, the skew will only occur on the x axis. There are also skewX() and skewY() available:

        transform: skew(15deg, 4deg);

      • TRANSITIONS:
        Animation has certainly been possible for some time with JavaScript, but native CSS transitions generally require less client-side processing, so they’ll usually appear smoother. On mobile devices with limited computing power, this can be a lifesaver.

        Here are the steps to create a simple transition using only CSS:

        1. Declare the original state of the element in the default style declaration.
        2. Declare the final state of your transitioned element; for example, a :hover state.
        3. Include the transition functions in your default style declaration using the transition properties, including: transition-property, transition-duration, transition-timing-function, and transition-delay. Example:
          .animatedSquare{
  width: 300px;
  height: 300px;
  background: #3DCCC7;
  transition-property: background, transform; //Notice that we didn't include the width change in the transition.
  transition-duration: 500ms; //It's best practice to write these in milliseconds
  transition-timing-function: ease-out;
  transition-delay: 500ms;
}
.animatedSquare:hover {
  width: 600px;
  background: #FF6B6B;
  transform: rotate(45deg);
}

        'transition:' is a shorthand (space sparated) for the transition properties property, duration, timing-function, delay but keep in mind the order.
        Shorthand Example for Single Property:
        transition: background 1000ms ease-in 250ms;
        The second number in the shorthand will always refer to the delay property.

        Shorthand Example for Multiple Properties:
        transition: background 1000ms ease-in 250ms, width 1000ms ease-out 250ms;
        Notice that multiple properties are comma separated.

        A good video tutorial on transition properties is here

        TRANSITION-DURATION:
        200ms is generally considered the optimum time for a transition: anything slower will make the website seem slow, drawing generally unwanted attention to what was supposed to be a subtle effect. Anything faster may be too subtle.

        TRANSITION-TIMING-FUNCTION:
        The most common timing functions include the key terms ease, linear, ease-in, ease-out, or ease-in-out. The default ease has a slow start, then it speeds up, and ends slowly.

    2. CANVAS

      • CHECK OUT THIS GUY'S VIDEOS ON CANVAS: CHRIS COURSES

        canvas was developed by Apple. It allows developers to draw shapes and lines, arcs and text, gradients and patterns. In addition, canvas gives us the power to manipulate pixels in images and even video. In order to user canvas you first must add the canvas tags like so:

        <canvas>
  Sorry! Your browser doesn't support Canvas.
</canvas>
        The text between the canvas tags will only show if the canvas element is not supported by the visitor's browser.

        Make sure you give your canvas a width and height HTML attributes, NOT IN CSS.
        Also, add a border since it's hard to find on page without one.         Example: 

        <canvas id="myCanvas" class="myCanvas" width="200" height="200"> 
  Sorry! Your browser doesn’t support Canvas.
</canvas>

.myCanvas {
  border: dotted 2px black;
}
