  1. The WINDOW Object:

    • Every JavaScript environment has a global object. Any variables that are created in the global scope are actually properties of this object, and any functions are methods of it. In a browser environment the global object is the window object, which represents the browser window that contains a web page.

    • Browser Object Model (BOM):

      The properties and methods of the BOM are made available through the window object. Every browser window, tab, popup, frame, and iframe has a window object.

      GLOBAL variables are variables that are created without using the const, let or var keywords. Global variables can be accessed in all parts of the program. Example:

      x = 6; // global variable created
OUTPUT: 6
window.x // same variable can be accessed as a property of the window object
OUTPUT: 6
// both variables are exactly the same
window.x === x;
OUTPUT: true

    • Browser Information:

      List of WINDOW object properties and methods here.

      WINDOW.LOCATION
      PROPERTIES:
      hash, host, hostname, href, origin, pathname, port, protocol, search
      METHODS:
      assign(), reload(), replace()

      WINDOW.HISTORY
      The window.history.length property shows how many pages have been visited before arriving at the current page.
      The window.history.go() method can be used to go to a specific page, where 0 is the current page:

      window.history.go(1); // goes forward 1 page
window.history.go(0); // reloads the current page
window.history.go(-1); // goes back 1 page

      window.history.forward(), window.history.back() Instead of referencing an specific URL for navigating forward and backwards on a webpage, if the user selects "Go Back" link on your page, for eaxmple, the window.history.back() method can be used within a Javascript event listener to allow the user to go back one page without having to code the specific URL of the page. [Although I think this isn't the best practice now that I think about various scenarios.]

    • Controlling Windows:

      Pop-ups can be created using the window.open() method. The window.open() signature looks like this: 

      window.open(URL, name, specs, replace).
      Examples:
      Blank window: var myWindow = window.open("", "", "width=200,height=100");
      New window: window.open("https://www.w3schools.com", "_blank", "toolbar=yes,scrollbars=yes,resizable=yes,top=500,left=500,width=400,height=400");

      The name parameter refers to the target of the window being opened. Options include
      "_blank" - URL is loaded into new window or tab.
      "_self" - URL replaces the current page.

      If replace parameter is set to true, then the new URL will replace the current URL in the browser's history. If set to false, then a new entry in the history list is created.

    • Screen Information:

      Some examples of uses are:

      // width and height will give total width and height of the screen.
window.screen.width;
window.screen.height;

// Available width and height will give dimensions minus any OS menu bars.
window.screen.availWidth;
window.screen.availHeight;

// Determine the screens color bit depth.
window.screen.colorDepth;

      The Screen object has more uses for mobile devices. It also allows you to do things like turn off the device's screen, detect a change in its orientation or lock it in a specific orientation.

    • The DOCUMENT Object:

      COOKIES:
      Cookies are small files that are saved locally on a userâ€™s computer. They are useful in that they allow HTTP data to be retained even though HTTP is a stateless protocol.
      Examples of this include a user's name being saved for site, whether he was previously logged in, or his unpurchased checkout items from his last visit.

      A restriction of cookies is that they can only be read by a web page from the same domain that set them. This is to stop sites being able to access information about users, such as other sites they have visited. Cookies are also limited to storing up to 4KB of data, although 20 cookies are allowed per domain, which can add up to quite a lot of data.

      Cookies are session cookies by default which means they are erased as soon as the browsing session is over (browser closed). But they can be made to persist even after browser closes by adding "; expires=date" when creating a cookie. the date portion is user determined. Example:

      const expiryDate = new Date();
const tomorrow = expiryDate.getTime() + 1000 * 60 * 60 * 24; expiryDate.setTime(tomorrow);
document.cookie = `name=Batman; expires=${ expiryDate.toUTCString()}`;

      An alternative is to set the max-age value. This takes a value in seconds:

      document.cookie = 'name=Batman; max-age=86400' // 86400 secs = 1 day
